# Project Documentation

This folder contains domain and architectural documentation for the e-commerce backend.

These documents describe **how the system works conceptually**, not just how it is implemented.

## Documents

- **purchase-flow.md**
  - Describes the complete purchase lifecycle:
    Cart → Checkout → Payment → Order
  - Defines domain responsibilities, state transitions, and business rules.
  - Acts as the domain contract for backend, frontend, and future features.

> Note: Operational and coding rules for contributors live in AGENTS.md.
